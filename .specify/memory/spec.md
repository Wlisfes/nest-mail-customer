# 财务预算管理系统 - 需求规范 (Spec)

> 本文档定义了系统的功能需求和用户期望，基于 MVP 产品设计文档整理。

## 一、产品概述

### 1.1 背景与痛点

公司每个季度或年度都会针对部门做预算，以前通过 Excel 表格进行管理，存在以下问题：

| 痛点 | 描述 |
|------|------|
| **费时费力** | 手工整理、汇总效率低 |
| **数据分散** | Excel 文件散落各处，难以统一管理 |
| **追溯困难** | 历史版本难以追踪，审批记录不清晰 |

### 1.2 产品目标

通过**财务预算管理系统**实现预算的数字化管控：

- ✅ 预算模板统一管理
- ✅ 预算在线提交与审批
- ✅ 预算数据集中存储与查询
- ✅ 审批流程可追溯
- ✅ 预算版本管理（V2.0）
- ✅ 操作日志审计（V2.0）

## 二、用户角色

### 2.1 角色定义

| 角色 | 说明 | 主要操作 |
|------|------|---------| 
| **财务管理员** | 财务部门人员，负责预算管理和审批 | 创建模板、指定上报部门、审批预算、查看所有预算 |
| **预算提交人** | 部门总监或指定员工，负责填报本部门预算 | 下载模板、上传预算、查看本部门预算 |

### 2.2 权限矩阵

| 功能 | 财务管理员 | 预算提交人 |
|------|-----------|-----------| 
| 创建预算模板 | ✅ | ❌ |
| 指定上报部门 | ✅ | ❌ |
| 下载模板 | ✅ | ✅ |
| 提交预算 | ✅ | ✅（仅本部门） |
| 审批预算 | ✅ | ❌ |
| 查看所有部门预算 | ✅ | ❌ |
| 查看本部门预算 | ✅ | ✅ |
| 移动端查看 | ✅ | ✅（仅本部门） |
| 预算汇总报表 | ✅ | ❌ |
| 操作日志查询 | ✅ | ❌ |

## 四、用户故事

### US-01: 财务管理员创建预算模板

```
作为 财务管理员
我希望 创建预算模板并指定需要上报的部门
以便 各部门按统一格式提交预算
```

**验收标准**:
- [ ] 可上传 Excel 模板文件（.xlsx/.xls）
- [ ] 可填写模板名称、预算周期、说明
- [ ] 可多选需要上报的部门
- [ ] 保存后自动推送钉钉通知给选定部门

### US-02: 预算提交人提交预算

```
作为 预算提交人
我希望 下载模板、填写后上传并提交审批
以便 完成本部门的预算上报
```

**验收标准**:
- [ ] 可查看并下载可用的预算模板
- [ ] 可上传填写好的 Excel 文件
- [ ] 上传后可在线预览确认内容
- [ ] 提交后状态变为"待审批"
- [ ] 自动推送通知给财务管理员

### US-03: 财务管理员审批预算

```
作为 财务管理员
我希望 在线查看部门预算并进行审批
以便 高效完成预算审核工作
```

**验收标准**:
- [ ] 可查看待审批列表
- [ ] 可在线查看 Excel 内容（无需下载）
- [ ] 可填写审批意见后通过或驳回
- [ ] 通过后状态变为"已定稿"
- [ ] 驳回后自动通知提交人

### US-04: 预算提交人查看版本历史

```
作为 预算提交人
我希望 查看预算的提交历史版本
以便 了解修改记录和变更内容
```

**验收标准**:
- [ ] 可查看预算的所有历史版本（V1, V2, V3...）
- [ ] 可点击查看任意版本的 Excel 内容
- [ ] 可看到每个版本的提交时间和变更说明

## 五、非功能需求

### 5.1 性能要求

| 指标 | 目标 |
|------|------|
| 页面首屏加载 | < 2s |
| API 响应时间 (P99) | < 500ms |
| 文件上传 (10MB) | < 5s |
| 并发用户数 | 100 |

### 5.2 安全要求

| 要求 | 说明 |
|------|------|
| 数据加密 | OBS 服务端加密存储 |
| 传输加密 | 全链路 HTTPS |
| 访问控制 | 基于角色的权限控制 |
| 审计日志 | 所有操作可追溯 |

### 5.3 可用性要求

| 指标 | 目标 |
|------|------|
| 系统可用性 | 99.9% |
| 数据备份 | 每日备份 |
| 故障恢复 | < 30分钟 |

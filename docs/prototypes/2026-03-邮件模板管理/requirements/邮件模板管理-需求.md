# 邮件模板管理 - 需求规格

## 基本信息

| 项目 | 值 |
|------|-----|
| **功能名称** | 邮件模板管理（拖拽式模板编辑器 + 模板发送 + 变量管理） |
| **所属迭代** | 2026-03-邮件模板管理 |
| **创建日期** | 2026-03-31 |
| **状态** | 草稿 |

## 需求背景

MailHub 邮件系统已具备基本的邮件收发功能，但用户每次发送格式化邮件（营销推广、活动邀请、周报等）都需要手动排版。引入 MJML 邮件模板系统后，用户可以通过拖拽组件快速构建响应式邮件模板，并支持变量替换群发，显著提升邮件运营效率。

**核心价值**：
- 降低格式化邮件的制作门槛（拖拽代替编码）
- 模板复用减少重复劳动
- 变量群发支持个性化批量沟通

## 目标用户

| 用户角色 | 特征 | 使用场景 |
|----------|------|----------|
| 邮件运营人员 | 需要频繁发送格式化邮件 | 创建营销模板、活动邀请，批量群发 |
| 普通用户 | 偶尔需要发送美观邮件 | 选择预设模板快速发送周报、通知 |

## 功能描述

### 核心功能

#### 1. 模板列表管理
- 卡片式展示模板列表，支持按分类筛选
- 模板 CRUD（新建 / 编辑 / 删除 / 复制）
- 模板分类管理（增删改）
- 模板预览（渲染 HTML 效果）
- 5 个系统预设模板（空白、营销推广、活动邀请、周报、通知）

#### 2. 拖拽式模板编辑器（三栏布局）

| 区域 | 内容 |
|------|------|
| **左侧 - 组件面板** | 8 类 MJML 组件可拖入画布 |
| **中间 - 画布区** | 拖拽排列组件，实时渲染预览 |
| **右侧 - 属性面板** | 编辑选中组件的属性（颜色、间距、文字等） |

**支持的 MJML 组件**：

| 组件 | 说明 | 可配置属性 |
|------|------|-----------|
| `mj-section` | 行容器 | 背景色、内边距、边框 |
| `mj-column` | 列布局 | 列数（1/2/3）、宽度比例、垂直对齐 |
| `mj-text` | 文本块 | 内容（CKEditor 5 富文本）、字体、颜色、对齐 |
| `mj-image` | 图片 | 图片URL（附件上传）、宽度、alt、链接 |
| `mj-button` | 按钮 | 文字、链接、背景色、圆角、内边距 |
| `mj-divider` | 分隔线 | 颜色、宽度、样式（实线/虚线） |
| `mj-social` | 社交链接 | 平台选择、链接URL、图标大小 |
| `mj-hero` | 大图横幅 | 背景图、标题文字、副标题、按钮 |

**编辑器功能**：
- 拖拽排序（vue-draggable-plus）
- 组件嵌套（section → column → 内容组件）
- 选中组件高亮 + 右侧属性面板联动
- 实时预览（MJML → HTML 编译）
- 撤销 / 重做
- 保存模板（MJML + HTML 双存储）

#### 3. 模板变量管理
- 变量 CRUD（变量名、变量键、默认值）
- 模板中通过 `{{变量键}}` 语法插入变量占位符
- 编辑器工具栏提供「插入变量」下拉菜单

#### 4. 模板发送（独立页面）
- 选择已有模板
- 预览模板内容
- 填充变量值（表单）
- 选择收件人（支持多个 → 群发）
- 群发时每个收件人可设置不同变量值（表格编辑）
- 预览最终效果后发送
- 通过现有 SMTP 发送通道投递

### 功能边界

- ✅ 包含：模板 CRUD、拖拽编辑器、8 类组件、分类管理、预设模板、变量管理、模板群发
- ❌ 不包含：定时发送、A/B 测试、发送数据统计、模板市场/共享

## 用户故事

- 作为**运营人员**，我希望**通过拖拽组件快速构建邮件模板**，以便**无需编写代码就能制作精美的邮件**
- 作为**运营人员**，我希望**在模板中插入变量并群发给多个收件人**，以便**每个人收到个性化的邮件内容**
- 作为**普通用户**，我希望**选择预设模板稍作修改后直接发送**，以便**快速完成周报、通知类邮件**
- 作为**用户**，我希望**对模板进行分类管理**，以便**快速找到需要的模板**
- 作为**用户**，我希望**管理模板变量**，以便**在不同模板中复用相同的变量定义**

## 业务规则

| 编号 | 规则 | 说明 |
|------|------|------|
| BR01 | 模板归属 | 模板为个人私有，只能看到自己创建的模板 |
| BR02 | 预设模板不可删除 | 系统预设模板只读，用户可复制后编辑 |
| BR03 | 分类关联 | 删除分类时，分类下的模板归入「未分类」 |
| BR04 | 双存储 | 保存模板时同时存储 MJML 源码和编译后的 HTML |
| BR05 | 变量替换 | 发送时系统将 `{{key}}` 替换为实际值，未填写的变量使用默认值 |
| BR06 | 群发隔离 | 群发时每个收件人独立发送，互不可见 |
| BR07 | 图片上传 | 使用现有附件上传接口，返回 URL 后插入模板 |

## 验收标准

- [ ] AC01: 模板列表页可按分类筛选，支持新建/编辑/删除/复制/预览
- [ ] AC02: 拖拽编辑器支持 8 类组件的拖入、排序、删除
- [ ] AC03: 选中组件后右侧属性面板可编辑属性，画布实时更新
- [ ] AC04: 文本块使用 CKEditor 5 编辑富文本内容
- [ ] AC05: 保存模板后数据库同时存有 MJML 和 HTML
- [ ] AC06: 5 个预设模板内置且不可删除，可复制使用
- [ ] AC07: 模板变量管理页支持 CRUD
- [ ] AC08: 编辑器可通过工具栏菜单插入 `{{变量}}` 占位符
- [ ] AC09: 模板发送页可选择模板、填充变量、选择多个收件人
- [ ] AC10: 群发时每个收件人的变量值可独立编辑
- [ ] AC11: 发送成功后收件人收到变量已替换的 HTML 邮件

## 边界与异常

| 场景 | 触发条件 | 处理方式 |
|------|----------|----------|
| MJML 编译失败 | 组件结构不合法 | 保存时提示错误，不允许保存 |
| 模板无内容 | 画布为空时点击保存 | 提示「请至少添加一个组件」 |
| 变量未填写 | 发送时有变量未填值 | 使用默认值替换，无默认值则留空 |
| 群发部分失败 | SMTP 部分投递失败 | 逐条标记状态，支持失败重发 |
| 删除已使用分类 | 分类下有模板 | 模板归入「未分类」 |
| 图片上传失败 | 附件接口异常 | 提示上传失败，保留原图片 URL |

## 非功能需求

| 类型 | 要求 |
|------|------|
| 性能 | 编辑器拖拽响应 < 100ms，MJML 编译 < 2s |
| 存储 | 单模板 MJML + HTML 预计 < 500KB |
| 兼容 | 编译后 HTML 兼容主流邮件客户端（Gmail、Outlook、QQ邮箱、163） |
| 安全 | 模板数据按 userId 隔离 |

## 数据模型

### tb_mail_template（邮件模板）

| 字段 | 类型 | 说明 |
|------|------|------|
| keyId | int (PK) | 主键 |
| userId | int (FK) | 所属用户 |
| categoryId | int (FK) | 分类ID |
| name | varchar(100) | 模板名称 |
| description | varchar(500) | 模板描述 |
| mjmlSource | mediumtext | MJML 源码 |
| htmlContent | mediumtext | 编译后 HTML |
| thumbnail | varchar(500) | 缩略图 URL |
| isPreset | tinyint | 是否预设（0否/1是） |
| createTime | datetime | 创建时间 |
| modifyTime | datetime | 修改时间 |

### tb_mail_template_category（模板分类）

| 字段 | 类型 | 说明 |
|------|------|------|
| keyId | int (PK) | 主键 |
| userId | int (FK) | 所属用户 |
| name | varchar(50) | 分类名称 |
| sort | int | 排序号 |
| createTime | datetime | 创建时间 |

### tb_mail_template_variable（模板变量）

| 字段 | 类型 | 说明 |
|------|------|------|
| keyId | int (PK) | 主键 |
| userId | int (FK) | 所属用户 |
| name | varchar(50) | 变量显示名（如「姓名」） |
| varKey | varchar(50) | 变量键（如 `name`） |
| defaultValue | varchar(500) | 默认值 |
| createTime | datetime | 创建时间 |

## 修复的 BUG

### BUG-001: 邮件详情页数据为空

- **发现于**: TC-003 首次执行
- **原因**: 后端 `httpFetchMailDetail` 返回 `{ data: { ...mail } }` 多嵌套了一层，前端 `res.data` 拿到的是包含 `data` 字段的对象
- **修复**: 去掉 `fetchResolver` 中多余的 `{ data: ... }` 包装
- **验证**: 修复后主题/发件人/时间正确显示

## 已知限制

1. **旧邮件无正文**: IMAP 同步拉取正文的代码是本次新增的，已存在的旧邮件 `htmlBody`/`textBody` 为 null。需要删除旧数据重新同步才能拉取正文。
2. **TC-005 未测试**: 真实 SMTP 发送需要验证 SMTP 凭证有效性，建议手动测试。
